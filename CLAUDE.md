# CLAUDE.md — bankrot-leads

Общие правила работы описаны в `/Users/mogr1m/Projects/CLAUDE.md` (действуют
и здесь: общение на русском, план → согласование → реализация, комментарии
везде, коммиты на русском и т.д.). Ниже — специфика именно этого проекта.

## Стек

- **Парсинг**: Python 3 (requests) — скрипты в `src/parsers/`
- **Сайт**: Next.js (App Router), задеплоен на Vercel через Git-интеграцию
  (автодеплой при пуше в `main`)
- **CRM для лидов**: готовое решение (AmoCRM/Bitrix24), свою не пишем —
  пока лиды уходят в Telegram-бота (`web/lib/leads.ts`)

## Данные, с которыми работаем

- В `data/` попадают выгрузки парсеров (контакты юрфирм, статистика дел).
  JSON-файлы (`data/*.json`) коммитятся в git — это публичные данные с
  2ГИС (название/адрес/рейтинг), сайт читает их напрямую из `data/` при
  сборке (`web/lib/organizations.ts`, `path.join(process.cwd(), "..", "data")`).
  CSV-выгрузки (`data/*.csv`) в git не идут — промежуточные/рабочие файлы
- Персональные данные людей, оставивших заявку (телефон, сумма долга),
  через парсеры не собираются — только через форму на будущем сайте.
  При работе с этими данными — обязательно 152-ФЗ (согласие, политика ПД)

## Юридическая осторожность

- Не писать в текстах сайта формулировки, гарантирующие результат
  ("спишем 100% долгов" и т.п.) — нарушение ФЗ "О рекламе"
- Перед реализацией парсеров kad.arbitr.ru и fedresurs.ru — сначала
  проверить вручную (в браузере), есть ли антибот-защита/капча, и уточнить
  у пользователя, прежде чем писать код обхода
